# Security policy

## Supported version

Security fixes are maintained for the current EdgeWatch release only.

| Version | Supported |
|---|---|
| 0.5.x | Yes |
| 0.4.x and older | No |

## Reporting

Do not open a public issue for a vulnerability that exposes authentication, secrets, host access, or private infrastructure data.

Send a private report to the repository owner with:

- affected version
- clear reproduction steps
- expected and actual behavior
- impact assessment
- logs or screenshots with secrets and private addresses removed
- suggested mitigation when known

Do not include Plex tokens, Linode tokens, ntfy topics, oauth2-proxy secrets, cookies, authorization headers, or full private snapshots.

## Response goals

This is a small self-hosted project.  Reports will be assessed as time permits.  A valid issue will be documented and fixed in a new release before public disclosure when practical.
